# wiki-research-scout — 6 种扫描模式检测指南

## 1. 高频被引用但内容薄弱

### 检测方法
- 从 `index.md` 中找出 related 数多的页面
- 遍历 wiki 页面，统计每页正文长度（不含 frontmatter）和 related 数量
- 对候选页读正文，判断内容深度

### 判断标准
```
正文 < 200 字              → 薄弱
正文 < 100 字 且 related > 5  → 严重薄弱
related > 3 但正文仅一句话    → 薄弱
```

### 建议动作
- 扩写页：补全定义、算法/原理、应用场景、实践注意事项
- 如果本身是个大主题 → 升级为新建专题页

---

## 2. 高频断链 / 待建概念

### 检测方法
- 从 health-check 报告中读 `single_direction_links` 列表
- 搜索 `[[...]]` 模式，找出：
  - 被多处引用但无对应 `.md` 文件的内容
  - 主要在 related frontmatter 中出现但无页面的内容
- 重点关注被引用 >3 次的概念

### 判断标准
```
被引用 ≥10 次                → 立即建议建页
被引用 5~9 次                → 中等优先级
被引用 2~4 次                → 低优先
仅 1 次                      → 忽略（可能是临场引用）
```

### 建议动作
- 新建页：建专题页概述概念 + 与引用它的所有页面互链
- 如果概念本身不重要（如一个人名）→ 标注后忽略

---

## 3. 单页过重，适合拆专题

### 检测方法
- 找出正文 > 1000 字的页面
- 判断正文是否包含多个可独立展开的章节
- 检查页面内容是否属于"综述/总览"类型

### 判断标准
```
正文 > 5000 字               → 明显过重，优先拆
正文 2000~5000 字且覆盖 3+ 子主题 → 可拆
正文 > 1000 字但结构单一      → 不拆
```

### 可拆的典型信号
- 页面标题含"综述""汇总""合集""大全"
- 有 5 个以上的 `##` 子节
- 文中大量 `[[...]]` 指向无页面概念
- 用户手动在该页内做了"目录跳转"

### 建议动作
- 拆页：为标准页 + 独立子专题页
- 新老页互链
- 老页保留摘要 + 指向子页的链接

---

## 4. 主题簇缺口

### 检测方法
- 发现一批相关页面之间缺少枢纽/总览页
- 已有多个"叶子页"但没有"根页"

### 典型例子
```
已有：因子A介绍、因子B介绍、因子C介绍
缺少：因子评价框架（总览页/枢纽页）

已有：VIP30、VIP31、SuperTrend、三色带
缺少：策略组合方法论（对比/选择框架）

已有：趋势跟踪、均值回归、套利
缺少：策略分类总览
```

### 判断标准
```
已有 ≥3 个同主题子页          → 考虑建枢纽页
子页之间在 related 中彼此引用了 → 已有自然簇，补枢纽页效果最好
子页之间互引 <2               → 簇未形成，不急于建枢纽
```

### 建议动作
- 新建枢纽页：为簇建立总览、对比、选择框架
- 建完后与所有子页互链

---

## 5. 跨主题交叉机会

### 检测方法
- 分析两个主题簇的页面是否在正文/related 中有共现
- 检查是否已有桥接页

### 典型例子
```
价格行为 (10 页) + 波动率 (5 页) → 交叉方向：价格行为在波动率过滤中的应用
VIP31 (7 页) + 多周期 (4 页) → 交叉方向：VIP31 的多周期信号融合
遗传规划 (8 页) + 因子挖掘 (3 页) → 交叉方向：遗传规划在因子挖掘中的应用
```

### 判断标准
```
两簇各有 ≥5 页                → 有足够基础
两簇之间已有共现              → 信任链形成
簇间无共现但语义关联          → 低优先级桥接机会
```

### 建议动作
- 新建桥接页：聚焦交叉点，不覆盖两簇原始领域
- 桥接页同时挂在两簇的 related 中

---

## 6. 陈旧页面需补新进展

### 检测方法
- 从 `log.md` 或文件修改时间中判断
- 对 6 个月内未更新的页面做标记
- 判断主题是否属于仍在演进的领域

### 判断标准
```
末次修改 > 6 个月 且主题在演进 → 建议查新
末次修改 > 12 个月             → 明确建议更新
末次修改 > 12 个月 且主题已稳定 → 可延后
```

### 可补新的典型主题
| 主题类型 | 演进速度 | 补新周期建议 |
|----------|----------|-------------|
| 量化论文 | 快（月级别） | 6 个月 |
| 策略框架 | 中 | 6~12 个月 |
| 价格行为 | 中 | 12 个月 |
| 交易心理 | 慢 | 12 个月以上 |
| 宏观框架 | 中 | 6~12 个月 |
| 基础概念 | 慢 | 18 个月以上 |

### 建议动作
- 扩写页：补充近 6~12 个月的进展
- 在页首添加"最后更新"标记
- 如果进展重大 → 考虑新建姊妹页

---

## 优先执行策略

做第一轮扫描时，按 `4 → 5 → 2 → 1 → 3 → 6` 的顺序寻找候选：

1. 先看**主题簇缺口**（先识别簇结构，为后续交叉做准备）
2. 再找**跨主题交叉**（簇结构已知后，找桥接机会，收益高）
3. 再看**高频断链**（痛点明显，修复效果直观）
4. 再看**内容薄弱页**（门槛低，1~2h 能搞定）
5. 再看**单页过重**（需判断拆与不拆的成本）
6. 最后看**陈旧页**（工作量最大，需查新资料）

实际写作时根据候选数量和用户偏好微调。
